# Skill: build-backend

Build the Python backend agent following the Elio platform conventions (contract v4).

**Your code must pass `back/tests/test_elio_contract.py`.** That file is the concrete rule source — read it before you write any code. It encodes every mechanical B-rule (import whitelist, `DeclarativeAgentApp` subclass, `@step` decoration, registration, test count, bilingual prompts, `interface_language` parameter, …) as real assertions. Prose guidelines (`.agents/docs/AGENT_APP_GUIDELINES_BACK.md` and `CONVENTIONS.md`) give context; the test file gives the truth.

## Build scope and the backlog ledger

`backlog.md` is both the plan and the progress ledger. **Default mode: build every unchecked Must Have in one invocation**, ticking each `**Status:**` box as that story's code + tests land. **Stop at the Must Have / Should Have boundary.** Never auto-promote Should or Could items — those require an explicit PO handoff.

The consultant can also invoke this skill in **single-story mode** by saying `iterate US-N` or `only US-N` — in that case build just that story and stop.

Procedure on every invocation:

1. **Read `backlog.md`.** Collect every US-N under `## Must Have` whose `**Status:**` line is `[ ] not started`. If none, stop and tell the consultant all Must Haves are done — they can promote a Should / Could via the PO, or invoke `package-agent`.
2. **For each Must Have, in backlog order:**
   a. Scope the work to that story only. Don't pull fields, routes, or steps from later stories.
   b. Build through the file-creation order below, limited to what this US-N requires. Earlier stories' files get extended rather than rewritten.
   c. Run the contract suite and the agent's own tests. Both must be green before moving on.
   d. **Tick the box.** Change `**Status:** [ ] not started` to `**Status:** [x] done — {short note}`. The tick IS the ledger — do not rely on git (the scaffold is transferred to consultants as a zip, not a repo). Tick **as soon as that story is green** — not at the end, so a partial run still leaves an honest ledger.
3. **Stop at the Must Have boundary.** Do not touch Should / Could stories. Emit the closing checklist (below) and wait for the consultant to invoke `build-frontend`, promote a Should, or run `verify-generation`.

If any story's tests fail and the 3-strike repair loop is exhausted, leave its box unticked with a `**Status:** [ ] blocked — {reason}` note, continue with the next Must Have that doesn't depend on it, and flag the blocker in the closing report.

## File creation order

Build files in this exact order. Each file must be syntactically correct before moving to the next.

**Test skeleton comes before `app.py`.** Step 3 writes `tests/test_app.py` with the 5 required test functions as `pytest.skip("TODO")` — B6 is then structurally guaranteed and you can't "forget" to write tests. Step 6 fills the bodies.

### 1. `back/agents/{name}/schemas.py`

`TabSchema` subclasses (Pydantic-backed tab state) + typed output models for `completed()` calls.

```python
"""Schemas for the {name} agent app."""

from pydantic import BaseModel, Field

from framework import TabSchema


class {Name}InputTab(TabSchema):
    """Input data provided by the user."""

    prompt: str = Field("", description="Main user request")
    context: str = Field("", description="Optional additional context")
    lang: str = Field("fr", description="Output language: 'fr' or 'en'")


class {Name}OutputTab(TabSchema):
    """Results generated by the agent steps."""

    # Step 1 result
    summary: str = Field("", description="Generated summary from Step 1")
    key_points: list[str] = Field(default_factory=list, description="Key points from Step 1")


class Step1Output(BaseModel):
    """Typed output for the step1 completed() call."""

    summary: str = Field(description="Generated summary")
    key_points: list[str] = Field(default_factory=list, description="Key points extracted")
```

Rules:
- All `TabSchema` fields **must have defaults** — empty state = initial state
- One `TabSchema` per tab defined in your API contracts
- Typed output models (`BaseModel`, not `TabSchema`) are only for `completed(data=...)` validation — they live in `schemas.py` alongside the tab schemas
- Google-style docstring on every class

### 2. `back/agents/{name}/prompts.py`

Single bilingual prompt file — no logic, constants only.

```python
"""Prompts for the {name} agent app (bilingual)."""

from typing import Literal


def get_step1_prompt(*, lang: Literal["fr", "en"], prompt: str, context: str = "") -> str:
    """Build the Step 1 system prompt."""
    context_block = f"\nContext:\n{context}" if context.strip() else ""
    if lang == "en":
        return f"""You are a consulting assistant at Onepoint.
User request: {prompt}{context_block}

[English instructions here]

Return a JSON object with exactly this structure (no markdown, no ```):
{{
  "summary": "...",
  "key_points": ["...", "..."]
}}

Respond ONLY with valid JSON."""
    return f"""Tu es un assistant consultant chez Onepoint.
Demande utilisateur : {prompt}{context_block}

[French instructions here]

Retourne un objet JSON avec exactement cette structure (sans markdown, sans ```):
{{
  "summary": "...",
  "key_points": ["...", "..."]
}}

Réponds UNIQUEMENT avec du JSON valide."""


def get_ui_message(lang: Literal["fr", "en"], key: str) -> str:
    """Return a localized UI progress message."""
    messages: dict[str, dict[str, str]] = {
        "init":        {"fr": "Initialisation...",          "en": "Initializing..."},
        "generating":  {"fr": "Génération en cours...",     "en": "Generating..."},
        "parsing":     {"fr": "Analyse de la réponse...",   "en": "Parsing response..."},
        "completed":   {"fr": "Terminé.",                   "en": "Done."},
    }
    return messages.get(key, {}).get(lang, key)
```

Rules:
- All user-facing strings live **only** in `prompts.py` — none in `app.py`
- Every prompt returns **only valid JSON** — include that instruction in the prompt text
- French prompts respond in French; English prompts respond in English

### 4. `back/agents/{name}/app.py`

```python
"""{Name} AgentApp — [short description].

Steps:
  1. step1  — [what it does]
  2. step2  — [what it does]  (if applicable)
"""

import json
import logging

from framework import DeclarativeAgentApp, SSEEvent, StepContext, TabSchema, step
from services.llm_config import get_llm

from .prompts import get_step1_prompt, get_ui_message
from .schemas import (
    {Name}InputTab,
    {Name}OutputTab,
    Step1Output,
)

logger = logging.getLogger(__name__)


class {Name}App(DeclarativeAgentApp):
    """{Name} agent app.

    [One-paragraph description of what the app does.]

    Tabs:
        {Name}InputTab: User inputs.
        {Name}OutputTab: Step results written progressively.
    """

    class Meta:
        tabs = [{Name}InputTab, {Name}OutputTab]

    @step(id="step1", persist_progress=True)
    async def step1(self, ctx: StepContext, inputs: {Name}InputTab, **kwargs):
        """[What this step produces].

        Args:
            ctx: StepContext with username and language.
            inputs: Validated input tab.
            **kwargs: Remaining flat state fields.

        Yields:
            SSEEvent dicts as text/event-stream data.
        """
        lang = inputs.lang or kwargs.get("lang", "fr")
        prompt = inputs.prompt or kwargs.get("prompt", "")
        context = inputs.context or kwargs.get("context", "")

        if not prompt.strip():
            yield self.error(
                step_id="step1",
                message=get_ui_message(lang, "empty_prompt"),
            )
            return

        yield self.in_progress(step_id="step1", message=get_ui_message(lang, "init"), progress=5)

        llm = get_llm()
        system_prompt = get_step1_prompt(lang=lang, prompt=prompt, context=context)

        yield self.in_progress(step_id="step1", message=get_ui_message(lang, "generating"), progress=30)

        response = await llm.ainvoke([("system", system_prompt)])
        raw = response.content.strip() if hasattr(response, "content") else str(response)

        yield self.in_progress(step_id="step1", message=get_ui_message(lang, "parsing"), progress=80)

        result = _parse_step1(raw, lang)

        yield self.completed(
            step_id="step1",
            message=get_ui_message(lang, "completed"),
            data=result.model_dump(),
        )


def _clean_json(raw: str) -> str:
    stripped = raw.strip()
    if stripped.startswith("```"):
        lines = stripped.split("\n")
        inner = lines[1:] if lines[0].startswith("```") else lines
        if inner and inner[-1].strip() == "```":
            inner = inner[:-1]
        return "\n".join(inner).strip()
    return stripped


def _parse_step1(raw: str, lang: str) -> Step1Output:
    try:
        data = json.loads(_clean_json(raw))
        return Step1Output(**data)
    except Exception:
        logger.warning("Step 1 JSON parse failed — using raw response as summary")
        return Step1Output(summary=raw[:500], key_points=[])
```

Rules:
- **Never add `@stream_safe`** — `@step` applies it automatically
- `from services.llm_config import get_llm` — never instantiate `AzureChatOpenAI` directly
- All user-facing strings come from `get_ui_message()` — no inline FR/EN strings
- `self.in_progress()` / `self.completed()` / `self.error()` — the only yield helpers
- `persist_progress=True` on the first step; `False` on subsequent steps (unless the step is a checkpoint)

## Banned patterns — learn from previous generations

Each of these is blocked by `back/tests/test_elio_contract.py`; avoiding them up front saves a repair loop.

| ❌ Banned | ✅ Correct |
|---|---|
| `async def {name}_step_1_stream(...)` standalone function | `class {Name}App(DeclarativeAgentApp)` with `@step` method |
| `@stream_safe` on a step method | Remove it — `@step` applies it automatically |
| `yield {"step": ..., "status": ..., "progress": ...}` raw dict | `yield self.in_progress(step_id=..., message=..., progress=...)` |
| `AGENTS_MAP["{name}-step-1"] = {name}_step_1_stream` | `REGISTERED_APPS = {"my-usecase": {Name}App}` in `registered_apps.py` |
| `from langchain_openai import AzureChatOpenAI; llm = AzureChatOpenAI(...)` | `from services.llm_config import get_llm; llm = get_llm()` |
| Model name not in `config_llms.json` | `get_llm()` (default) or `get_llm("gpt-5-chat")` |
| Hardcoded FR/EN strings in `app.py` | All strings via `get_ui_message(lang, key)` from `prompts.py` |
| Missing `lang` / `interface_language` handling | Read `lang` from `inputs.lang` or `kwargs.get("lang", "fr")` |
