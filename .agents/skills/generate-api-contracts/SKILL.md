# Skill: generate-api-contracts

Write the API contract and shared TypeScript types before any backend or frontend code.
This is the single source of truth that both build skills will follow.

---

## Prerequisites

- `intake-from-markdown` has been confirmed by the user
- `replit.md` "App being built" section is filled in

---

## Step 1 — Write `docs/api-contracts.md`

Create or overwrite `docs/api-contracts.md`.

For each agent step, document:

```markdown
## Step N — {Step Name}

**Route:** POST /agent-apps/{agent-id}-step-N/stream
**Content-Type:** application/json

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| username | string | yes | Requester username |
| interface_language | string | no | "fr" or "en" (default "fr") |
| [field] | [type] | [yes/no] | [description] |

### SSE event stream

| Event (step) | Status | Progress | Payload |
|--------------|--------|----------|---------|
| beginning | in_progress | 0 | — |
| generating | in_progress | 30–80 | — |
| completed | completed | 100 | result: {Step N Result} |
| error | error | 0 | error: string |

### Result shape

```json
{
  "[field]": "[type] — [description]"
}
```

### Example request

```json
{
  "username": "consultant@onepoint.fr",
  "prompt": "...",
  "interface_language": "fr"
}
```

### Example SSE sequence

```
data: {"step":"beginning","message":"Initialisation...","status":"in_progress","progress":0}
data: {"step":"generating","message":"Génération en cours...","status":"in_progress","progress":50}
data: {"step":"completed","message":"Terminé !","status":"completed","progress":100,"result":{...}}
```
```

## Step 2 — Write `packages/shared-types/src/index.ts`

Create or overwrite the file with TypeScript interfaces matching the contract.

```typescript
/**
 * Auto-generated by generate-api-contracts skill.
 * Matches back/agents/{name}/models.py — regenerate if models change.
 */

// ── Step 1 ────────────────────────────────────────────────────────────────────

export interface {Name}Step1Input {
  username: string;
  prompt: string;
  interface_language?: "fr" | "en";
  // ... all input fields
}

export interface {Name}Step1Result {
  // ... all output fields with JSDoc comments
}

// ── Step 2 (if applicable) ────────────────────────────────────────────────────

export interface {Name}Step2Input {
  username: string;
  prompt: string;
  step1_result: {Name}Step1Result;
  interface_language?: "fr" | "en";
}

export interface {Name}Step2Result {
  // ... all output fields
}
```

Rules:
- Use `string`, `number`, `boolean`, `string[]`, or nested interfaces — no `any`
- Add JSDoc comment on every field explaining what it contains
- Export every interface (all are public)
- Keep field names identical to the Python Pydantic models (snake_case)

## Step 3 — Update `replit.md`

Set "API contracts" status to ✅ in the build progress table.
Fill the "API surface" field in "App being built".

Show the user a summary:

```
Contrats API générés :
- docs/api-contracts.md → [N] routes documentées
- packages/shared-types/src/index.ts → [N] interfaces TypeScript

Je commence le backend ? (oui / non)
```

Wait for confirmation before invoking `build-backend`.
